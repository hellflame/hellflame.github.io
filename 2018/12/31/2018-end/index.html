在即将结束的2018年

发布 : 2018-12-31 分类 : diaries 浏览 : --

时光飞逝，不知不觉已经是2018年酷爱结束了，就着整理博客网站的契机，更新一篇 2018年 的总结

Python

python使用越来越熟练，自从当起面试官，有一种可以在简历上写下 Python 精通 的错觉

对于 元类 的使用也日趋成熟。动态篡改老旧代码，实现批量功能的技术也是用的让自己害怕=。= (不过还是喜欢用，尤其是在为老旧代码换上新衣的时候)

上半年基本上是轻松的半年，公司旅游、任务简单，还积累了很多实践

下半年开始着手 微服务 基础设施搭建，服务的开发框架也开始在这个时候定型。完成了 tornado、express、socketio 等框架建设，docker、etcd 等基础设施上线，感觉所有东西都要自己动手。。。

在年底最后一天的两个小时里，完成了 接口测试框架开发(顺便写完了一级页面接口的测试配置)，将测试配置翻译为测试用例，还差一步就可以写成新的动态语言了(不过在明年这可能就是另一篇文章了)，为自己的开发能力感到骄傲和口怕。

总而言之，python 是一门不错的动态语言、建模语言，使用它可以在极短时间内完成模型开发和动态修改，是一匹容易驾驭的宝马，同时一个随意的项目开端必定需要持续的重构，技术债务的堆积速度可能超乎想象，后期维护将是一个大问题，是一匹容易脱缰的马

Golang

作为强类型语言的选择，golang 能够避免很多动态语言的问题，工程化能力更加健壮，平台移植性也不是问题。以后会有很多项目采用 golang 开发

语言其实从来都不是问题的关键，重要的是如何使用语言以及使用者
